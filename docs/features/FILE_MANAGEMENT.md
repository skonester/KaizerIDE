# File Management

KaizerIDE includes local and remote file workflows for browsing, searching, opening, and editing project files.

[← Back to features](../FEATURES.md) · [Docs hub](../README.md)

---

## File explorer

The sidebar file explorer lets you browse and manage project files from the active workspace.

Key capabilities include:

- **Tree browsing** - Navigate project folders and files.
- **File operations** - Open, manage, and organize files from the workspace UI.
- **Drag and drop** - Move through common file workflows quickly.
- **Context menus** - Access file and folder actions from the explorer.

---

## Search panel

KaizerIDE includes fast file and text search across your workspace so you can locate files, symbols, and content without leaving the IDE.

Workspace indexing can make search and AI context discovery faster. See **[Workspace Indexing System](INDEXING.md)** for details.

---

## `.gitignore` awareness

The file tree honors your repository `.gitignore` rules automatically, keeping ignored files and generated directories from cluttering day-to-day navigation.

---

## Non-blocking tree reads

Directory reads are asynchronous and concurrent, reducing UI freezes on large repositories and improving startup responsiveness.

---

## SSH/SFTP support

KaizerIDE can connect to remote servers through SSH/SFTP so you can browse and edit remote files directly from the desktop app.

---

## Windows Explorer integration

KaizerIDE supports right-click context menu integration from Windows Explorer so files and folders can be opened directly in the IDE.
